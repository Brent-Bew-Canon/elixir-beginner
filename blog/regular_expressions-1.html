Regular Expressions 1: Decoding the Code

Like what the heck is this stuff? I almost gave up on coding when I first saw these. However, with practice and patience, I can read and use them now.

Don’t get me wrong, when you place me in front of a regular expression it takes me several minutes to digest what is actually going on still.

Let’s start with a complex example right out of the gate:

My_reg_expression = ~r{\/(\w+)\?id=(\d+)}

Don’t panic! I’m going to walk you through this step-by-step. First of all, the goal of this particular regular expression is to capture the path of a URL that will look like one of these:

/blog?id=2

/site?id=3

/pumpkin?id=8

/whatever?id=0

Get the pattern? Let’s highlight the characters that actually mean what they look like:

My_reg_expression = ~r{\ / (\w+)\?id=(\d+)}

/blog ?id=2

So these highlighted characters are what they are. In other words, the expression is looking for the URL and pulling those exact characters out.

Now we can safely ignore those characters knowing what they are. Next, let’s move on to another easy part.

My_reg_expression = ~r{\/(\w+)\?id=(\d+)}

The ~ combined with the r is called a sigil. That’s simply how we tell Elixir to run this as a regular expression (regex). The curly braces {} contain the regular expression. The parentheses () are also a simple character that just groups whatever’s inside them together. This will make more sense later, but it’s really that simple.

Just like that we are half-way there! Now, the more complicated pieces:

My_reg_expression = ~r{\/(\w+)\?id=(\d+)}

This is where it gets a little more tricky. The \w means look for normal characters like a,b,c or 1,2,3. That special combo of \w represents that functionality.

The \d is similar, but only looks for numbers like 1,2,3.

Finally, the + sign at the end of both of those means to look for one or more values. Alternatively, omitting the + would return just one character.

The last piece is to understand the final two backslashes:

My_reg_expression = ~r{\/(\w+)\?id=(\d+)}

We talked about in the very first step how there were some characters that we simply wanted to pull directly out of the URL without any special modifications or parameters.

These backslashes signal to do that. So really, I’d highlight it like this:

My_reg_expression = ~r{\/(\w+)\?id=(\d+)}

So each highlighted backslash is telling Elixir that the next characters in the expression should be taken literally instead of being used as a code for something else. Thus, we get the forward slash / and the ?id= as results from our expression.

That wasn’t so bad right? This was just one example of what regular expressions look like and can do. Take a minute to look it over again and comprehend what each character is doing.

My_reg_expression = ~r{\/(\w+)\?id=(\d+)}

Obviously, it’s vital to know what each symbol and symbol combination means if you want to get anywhere with regular expressions. Each programming language may use a different “flavor” of regexes, but most have similar components.

Elixir is based heavily on Perl programming regular expressions. I would suggest studying a document like this cheat sheet: https://www.geeksforgeeks.org/perl-regex-cheat-sheet/

Hope this helped! I’ll be practicing regular expressions from here-on-out. Let me know if this helped you and if you’d like to see more content on this topic.
