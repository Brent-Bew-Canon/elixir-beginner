Using the Pipe Operator: Key for Cleaner Code

Here’s a post on a handy little trick that you will use and see all over Elixir: the pipe operator “ |> “

The pipe operator is a shortcut that saves you some keystrokes and cleans up your code to look a lot nicer.

The way the pipe operator is used is when calling functions within functions. You can always call functions the classic way by using their name and passing in the arguments like so:

def special_function() do

a = 5

b = 6

c = 5 + 6

my_function(c)

end

The function special_function assigns values of a and b and then adds them together with the assignment for c. Then, it calls another function, my_function, and passes the c argument in.

However, there is a cleaner way to write this code with the pipe operator:

def special_function() do

a = 5

b = 6

c = 5 + 6

|> my_function()

end

Essentially, the |> is pulling the value from the line above and plugging it into the function. In this case that is c. It’s important to note that c is passed in as the first argument to the function and it is not necessary to type c in as an argument.

This doesn’t seem to be all that shorter or cleaner with this simple example. Let’s imagine that we have multiple functions like so:

def special_function() do

a = 5

b = 6

c = 5 + 6

|> my_function()

|> function_2()

|> function_3(b)

|> function_4(a,b)

end

Now you can start to see how this operator helps save space and keystrokes. One thing to notice now is that we have typed in an argument for function_3 and two arguments for function_4.

Knowing what we know about the pipe operator, we can tell that those functions must require multiple arguments. For example, function_3 is receiving whatever the output is that is returned from function_2 AND is also receiving the argument of b.

Since the pipe operator passes in the first argument automatically, any additional arguments still need to be passed in manually like we just discovered.

Now you know how to use the pipe operator to write cleaner Elixir code when dealing with multiple functions.
